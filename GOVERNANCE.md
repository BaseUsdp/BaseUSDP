# Governance

## Overview
BaseUSDP is governed by a combination of core team leadership and community input. As the project matures, governance will progressively decentralize.

## Current Governance Structure

### Core Team
The core team makes day-to-day decisions about protocol development, security, and operations. Core team members are listed in MAINTAINERS.md.

### Community Input
- **GitHub Discussions:** Feature proposals and feedback
- **GitHub Issues:** Bug reports and improvement suggestions
- **Community Calls:** Monthly open calls for project updates and Q&A

## Decision-Making Process
1. **Proposals:** Anyone can submit a proposal via GitHub Discussions
2. **Discussion:** Community discusses the proposal for at least 7 days
3. **Review:** Core team reviews feasibility and security implications
4. **Decision:** Core team makes the final decision, with rationale documented
5. **Implementation:** Approved proposals are added to the roadmap

## Future Governance
As the protocol matures, we plan to introduce:
- Token-based voting for protocol parameter changes
- Community-elected council members
- On-chain governance for treasury management
- Formal improvement proposal process (BIPs — BaseUSDP Improvement Proposals)

## Transparency
All governance decisions are documented publicly in the repository. Meeting notes from community calls are published in docs/governance/.

## Code of Conduct
All participants in BaseUSDP governance must adhere to our Code of Conduct.
