# BASEUSDP Roadmap

This roadmap outlines the planned development milestones for BASEUSDP. Priorities may shift based on community feedback and ecosystem developments.

## Completed

### v0.1.0 — Foundation (Feb 2026)
- [x] Dashboard with encrypted balance display
- [x] Send/receive payments with address resolution
- [x] ZK proof transfer system (three privacy tiers)
- [x] x402 payment protocol integration
- [x] Transaction history with CSV export
- [x] XMTP encrypted messaging
- [x] Multi-wallet support (MetaMask, Coinbase Wallet, Phantom)

### v0.2.0 — Public Launch (Mar 2026)
- [x] Landing page with feature showcase
- [x] Open-source release under MIT license
- [x] Public documentation and README

### v0.3.0 — Community (Mar 2026)
- [x] Contributing guidelines
- [x] Issue and PR templates
- [x] Security policy
- [x] Architecture documentation

### v0.4.0 — Name Resolution (Mar 2026)
- [x] ENS name resolution (alice.eth)
- [x] Base Name resolution (alice.base)
- [x] useResolveName React hook with caching

### v0.5.0 — Developer Experience (Mar 2026)
- [x] Test suite with Vitest
- [x] CI/CD with GitHub Actions
- [x] i18n support (English, Spanish, French, Chinese, Arabic)
- [x] Type system improvements
- [x] Performance optimizations
- [x] SEO improvements

## In Progress

### v0.6.0 — Batch Transfers (Target: Apr 2026)
- [ ] Send payments to multiple recipients in one transaction
- [ ] CSV upload for batch payments
- [ ] Gas optimization for batched transfers
- [ ] Batch transfer history view

### v0.7.0 — Payment Requests (Target: Apr 2026)
- [ ] Request payment from any wallet address
- [ ] Shareable payment request links
- [ ] QR code generation for in-person payments
- [ ] Payment request notification via XMTP

## Planned

### v0.8.0 — Mobile Optimization (Target: May 2026)
- [ ] Progressive Web App (PWA) support
- [ ] Mobile-optimized transaction flows
- [ ] Push notifications via service worker
- [ ] Offline transaction queuing

### v0.9.0 — Analytics Dashboard (Target: May 2026)
- [ ] Spending analytics and categorization
- [ ] Privacy score tracker
- [ ] Gas fee history and optimization suggestions
- [ ] Export to accounting software formats

### v1.0.0 — Production Release (Target: Jun 2026)
- [ ] Security audit by third-party firm
- [ ] Gas sponsorship for new users
- [ ] Fiat on-ramp integration (Coinbase Pay, MoonPay)
- [ ] Multi-chain support (Optimism, Arbitrum)
- [ ] Hardware wallet support (Ledger, Trezor)

## Future Considerations

- **Account abstraction** — Smart contract wallets with social recovery
- **Cross-chain privacy** — ZK bridges between L2s
- **Recurring payments** — Subscription and scheduled transfers
- **DAO treasury management** — Privacy-preserving multi-sig
- **SDK release** — npm package for third-party integration
