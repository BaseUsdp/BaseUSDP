# PR Review Checklist

## Code Quality
- [ ] Readable, well-structured
- [ ] Follows coding standards
- [ ] No debugging code

## Functionality
- [ ] Works as described
- [ ] Edge cases handled
- [ ] No regressions

## Testing
- [ ] New tests added
- [ ] All tests pass

## Security
- [ ] No secrets exposed
- [ ] Input validation present
- [ ] Auth checks in place

## Performance
- [ ] No unnecessary re-renders
- [ ] Bundle size acceptable

## Documentation
- [ ] APIs documented
- [ ] Changelog updated if needed
