# SEO Guide

This document covers the SEO strategy and implementation for BASEUSDP.

## Meta Tags

Every public page should use the `SEOMeta` component:

```tsx
import { SEOMeta } from "@/components/SEOMeta";

function SwapPage() {
  return (
    <>
      <SEOMeta
        title="Swap Tokens"
        description="Swap ETH, USDC, and other tokens on Base with low fees."
        canonical="https://baseusdp.com/swap"
      />
      {/* Page content */}
    </>
  );
}
```

### Authenticated Pages

Dashboard and other authenticated pages should use `noIndex`:

```tsx
<SEOMeta title="Dashboard" noIndex />
```

## Structured Data

Add JSON-LD structured data for rich search results:

```tsx
import { StructuredData, BASEUSDP_ORG_SCHEMA } from "@/components/StructuredData";

// In your root layout or landing page:
<StructuredData schema={BASEUSDP_ORG_SCHEMA} />
```

### FAQ Schema

```tsx
<StructuredData
```

## Sitemap

The sitemap is generated by `scripts/generate-sitemap.ts`. Run it as part of the build:

```bash
npm run build && npx tsx scripts/generate-sitemap.ts
```

The output includes:
- `dist/sitemap.xml` — All public URLs with priorities and change frequencies
- `dist/robots.txt` — Crawl directives and sitemap location

## Canonical URLs

Use the `canonicalUrl()` utility to generate consistent URLs:

```typescript
import { canonicalUrl } from "@/utils/seo";

canonicalUrl("/swap");          // "https://baseusdp.com/swap"
canonicalUrl("/swap?ref=x");    // "https://baseusdp.com/swap"
canonicalUrl("/swap/");         // "https://baseusdp.com/swap"
```

## Social Sharing

### Open Graph

All pages include Open Graph tags for Twitter, Discord, Telegram, and other platforms that render link previews.

Required tags:
- `og:title` — Page title
- `og:description` — Page description (max 200 chars)
- `og:image` — Social preview image (1200x630 recommended)
- `og:url` — Canonical URL

### Twitter Card

We use `summary_large_image` cards for maximum visual impact in tweets.

## Best Practices

1. Every public page must have a unique `<title>` and `<meta description>`
2. Use semantic HTML (`<h1>`, `<nav>`, `<main>`, `<article>`)
3. Add `alt` text to all images
4. Ensure all links have descriptive text (no "click here")
5. Keep meta descriptions between 120-160 characters
6. Use one `<h1>` per page
7. Internal links should use `react-router-dom` `<Link>` for SPA navigation
