# Internationalization (i18n)

BASEUSDP supports multiple languages through a lightweight, custom i18n system.

## Supported Languages

| Code | Language | Direction | Status |
|------|----------|-----------|--------|
| `en` | English | LTR | Complete |
| `es` | Spanish | LTR | Complete |
| `fr` | French | LTR | Complete |
| `zh` | Chinese (Simplified) | LTR | Complete |
| `ar` | Arabic | RTL | Complete |

## Architecture

The i18n system consists of three parts:

1. **Configuration** (`src/i18n/config.ts`) — Locale definitions, browser detection, and storage
2. **Locale files** (`src/i18n/locales/*.json`) — Translation strings organized by namespace
3. **React hook** (`src/hooks/useTranslation.ts`) — Dynamic loading and translation function

### Translation Keys

Keys use dot notation to navigate nested JSON structures:

```typescript
const { t } = useTranslation();

t("common.loading");        // "Loading..."
t("transfer.title");        // "Send Payment"
t("errors.networkError");   // "Network error. Please check your connection."
```

### Interpolation

Pass dynamic values using double curly braces:

```typescript
// In locale file: "greeting": "Hello, {{name}}!"
t("greeting", { name: "Alice" }); // "Hello, Alice!"
```

## Usage

### Basic Translation

```tsx
import { useTranslation } from "@/hooks/useTranslation";

function MyComponent() {
  const { t } = useTranslation();

  return <h1>{t("dashboard.title")}</h1>;
}
```

### Language Switching

```tsx
import { LanguageSwitcher } from "@/components/LanguageSwitcher";

function Settings() {
  return <LanguageSwitcher showNative />;
}
```

### Programmatic Locale Change

```tsx
const { setLocale } = useTranslation();
setLocale("es"); // Switch to Spanish
```

## Adding a New Language

1. Create a new locale file in `src/i18n/locales/` (e.g., `de.json`)
2. Copy the English file and translate all values
3. Add the locale to `SUPPORTED_LOCALES` in `src/i18n/config.ts`
4. Add the dynamic import in `src/hooks/useTranslation.ts`

## RTL Support

Arabic (and any future RTL languages) automatically set `dir="rtl"` on the document root when selected. Use the RTL utilities in `src/utils/rtl.ts` for direction-aware styling.

## Locale Detection

The system detects the user's language in this order:

1. Stored preference in `localStorage` (`baseusdp_locale`)
2. Browser `navigator.languages` / `navigator.language`
3. Default: English (`en`)
