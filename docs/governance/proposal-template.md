# Proposal Template

Use this template when submitting a BaseUSDP Improvement Proposal (BIP).

---

## BIP-[NUMBER]: [Title]

**Author:** [Your name or handle]
**Status:** Draft | Discussion | Voting | Approved | Rejected
**Created:** [Date]

### Summary
A brief (2-3 sentence) description of the proposal.

### Motivation
Why is this proposal needed? What problem does it solve?

### Specification
Detailed technical description of the proposed change.

### Rationale
Why was this approach chosen over alternatives?

### Alternatives Considered
What other approaches were considered and why were they rejected?

### Security Considerations
How does this proposal affect the security of the protocol?

### Privacy Implications
Does this proposal affect user privacy? If so, how?

### Implementation Plan
- Estimated timeline
- Required resources
- Testing plan
- Deployment strategy

### References
Links to relevant discussions, research, or prior art.

---

## Submission Process
1. Copy this template
2. Fill in all sections
3. Submit as a GitHub Discussion in the "Proposals" category
4. Allow 7 days for community discussion
5. Core team will schedule a vote if the proposal gains sufficient support
