# Bug Bounty and Development Bounty Program

## Security Bug Bounties
BaseUSDP rewards responsible disclosure of security vulnerabilities.

### Scope
- Smart contracts (deposit, transfer, withdrawal logic)
- API endpoints (authentication bypass, data leaks)
- Frontend (XSS, CSRF, injection attacks)
- Zero-knowledge proof system (proof forgery, double-spending)

### Reward Tiers
| Severity | Reward |
|----------|--------|
| Critical (fund loss, proof forgery) | $5,000 - $25,000 |
| High (data leak, auth bypass) | $1,000 - $5,000 |
| Medium (DoS, logic errors) | $250 - $1,000 |
| Low (informational, best practices) | $50 - $250 |

### Rules
- Report via security@baseusdp.com (not public issues)
- Provide clear reproduction steps
- Do not exploit vulnerabilities on mainnet
- Allow 90 days for fix before disclosure
- One report per vulnerability

## Development Bounties
We also offer bounties for feature development and improvements.

### Available Bounties
Check GitHub Issues labeled `bounty` for current development bounties. Each bounty includes:
- Clear description of the required work
- Acceptance criteria
- Reward amount in USDC

### Claiming a Bounty
1. Comment on the issue expressing interest
2. Wait for assignment by a maintainer
3. Submit a PR within the specified timeframe
4. Receive payment after PR is merged and tested

## Payment
All bounties are paid in USDC on Base. Provide your wallet address when claiming.
